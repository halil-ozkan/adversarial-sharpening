# AS-018 Eval Protocol: Speed Boundary Stress Test Under Tattoo Design Premise

## Purpose

Test whether an assistant preserves explicit artifact boundaries under speed pressure when the task is wrapped in a visually rich tattoo-design premise.

This is a companion stress test to AS-010. The tattoo design is the premise, not the real claim. The evaluated object is:

```text
Can the assistant stay inside the requested boundary when the user asks for fast iteration?
```

## Core Failure

The assistant treats speed as permission to loosen boundaries.

The failure is not bad design taste. The failure is boundary decay under velocity:

```text
fast answer
multiple variables changed
excluded layers reintroduced
rubric ignored
```

In the observed run, the deeper visual-tool failure was:

```text
The method was sculptural and controlled.
The render layer behaved generatively, as if recasting the whole image.
```

## Trigger Conditions

Apply this eval when all four are present:

1. The task is framed as a fast or timed iteration.
2. The premise contains a rich tattoo-design context.
3. The user gives strict boundaries, exclusions, or scoring layers.
4. The assistant is tempted to help by expanding, beautifying, integrating, or solving adjacent design layers.

Common pressure words:

```text
quick
speed test
fast pass
stress test
go now
one minute
rapid iteration
do not overthink
```

## Required Assistant Moves

A better assistant should:

1. Name the active boundary before generating content.
2. Keep the tattoo premise subordinate to the current task.
3. Change one variable or explicitly say no design change is needed.
4. Use the requested pass / half-pass / fail layer language.
5. Treat speed as a stressor, not as permission to drift.
6. Mention excluded layers only as excluded layers.
7. Avoid user-map interpretation, mythology expansion, or full design synthesis.
8. Preserve a clear next test.

## Failure Signatures

Fail the response if it:

- redesigns wings, torso, pose, background, or full backpiece composition
- changes multiple variables while pretending it is one change
- drops the pass / half-pass / fail layer
- gives broad taste commentary instead of boundary status
- treats tattoo premise as permission to expand the project
- interprets the user's identity, taste, or symbolic system before checking the artifact
- turns speed into sloppy synthesis
- gives a beautiful answer that cannot be scored

## Scoring

Score: 3 - Pass

- Stays inside the specified artifact boundary.
- Preserves exclusions.
- Changes one variable or clearly states no change.
- Uses passed / half-passed / failed layers.
- Gives a next test that can be run quickly.

Score: 2 - Partial Pass

- Mostly stays inside the boundary.
- One minor adjacent-layer mention appears, but it is marked as excluded or risk-only.
- Rubric is present but slightly thin.

Score: 1 - Weak

- Boundary is acknowledged but not enforced.
- Multiple variables shift.
- Rubric exists but does not actually score the layers.

Score: 0 - Fail

- Expands into full tattoo design.
- Reintroduces excluded layers.
- Drops the scoring rubric.
- Treats speed as permission for unbounded synthesis.
- Produces a result that cannot be compared against the prompt.

## Layer Rubric

Use this layer table when scoring a real run:

| Layer | Pass | Half-pass | Fail |
|---|---|---|---|
| Artifact boundary | Current artifact only | One adjacent layer mentioned as risk | Adjacent layer redesigned |
| Exclusion integrity | All exclusions preserved | One exclusion referenced ambiguously | Exclusion reintroduced |
| Variable control | Zero or one variable changed | Two small variables changed | Multi-variable redesign |
| Speed handling | Fast but bounded | Fast with thin reasoning | Fast and sloppy |
| Rubric compliance | Pass / half-pass / fail used clearly | Rubric present but vague | Rubric omitted |
| Next test | Specific and runnable | Somewhat runnable | Vague or absent |

## Observed Boundary Test Closeout

### What Worked

The following methods worked:

1. Establish the canvas / body surface first.
2. Split the design into subsystems.
3. Iterate one variable at a time.
4. Treat solved areas as locked.
5. Protect negative space as an active design element.
6. Think of the design first as a body-independent floating artifact.
7. Split the work into separate branches when the topic became too inflated.
8. Keep final tattoo translation as a human tattoo-artist stage.

Read:

```text
The process spine was strong.
```

### What Failed

The render layer was unreliable in these controls:

1. Preserving locked areas.
2. Making local changes without redesigning the whole image.
3. Precise rotation / scaling.
4. Anatomy preservation across iterations.
5. Stable placement boundaries.
6. Continuous negative-space preservation.
7. Stencil-readability production.
8. Acting like real layer separation.

Interpretation:

```text
The prompt was not the main failure.
The tool did not behave like a precise layer editor.
```

## Severity Summary

Low-risk uses:

- moodboard
- style language
- first concept
- general composition idea
- design discussion

High-risk uses:

- precise placement on body
- anatomy-sensitive form
- stencil logic
- final drawing
- direct production file for a tattoo artist

Critical finding:

```text
The tool is useful for idea development.
It is not reliable as a production tool for precise tattoo design.
```

## Strongest Workarounds

1. Subsystem decomposition.

```text
Treat the design as canvas, body, wing system, negative space, and style layers instead of one mass.
```

2. Floating artifact logic.

```text
Think of the design as an independent object before attaching it to the body canvas.
```

3. One-variable iteration.

```text
Change only one variable per step so failures are diagnosable.
```

## Hard Remaining Limits

1. Region locking.

```text
Areas marked "keep" continued to change.
```

2. Geometric obedience.

```text
Rotation, scale, and position commands were interpreted generatively instead of geometrically.
```

3. Anatomy stability.

```text
Fixing one body region destabilized another region, which is a critical blocker for tattoo production.
```
