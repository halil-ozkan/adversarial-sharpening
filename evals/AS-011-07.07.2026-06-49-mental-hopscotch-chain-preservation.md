# AS-011 Eval Protocol: Mental Hopscotch Chain Preservation

## Purpose

Test whether an assistant can preserve a useful cross-domain associative chain without flattening it into random topic drift or inflating it into identity mythology.

This is a companion eval to AS-011 because it operationalizes the user's "hopscotching / traversing a river" continuation note: flexible lane changes can work when the assistant tracks the invariant between tiles instead of forcing a rigid single lane.

## Core Failure

The assistant treats a cross-domain jump as either:

- random topic drift with no structure
- an excuse for inflated personal mythology
- a prompt to over-classify the user's mind
- a reason to force the conversation back into a rigid lane

## Correct Behavior

A better assistant should:

1. Parse named things literally first.
2. Identify the repeated emotional or structural invariant.
3. Map how the invariant changes across domains.
4. Keep the claim bounded.
5. Preserve the user's flow without over-validating every association.
6. Ask for the linking property only when the invariant is not visible.

## Trigger Conditions

Apply this eval when a user makes a quick cross-domain chain involving media, music, film, visual scenes, concepts, or memories, especially when the user frames it as a jump, tile, hopscotch, bridge, or lane shift.

Examples:

```text
Nas -> Gotham -> Massive Attack
street pressure to city pressure to nervous-system pressure
this is mental hopscotch
same weather system, different medium
```

## Target Response

```text
This is a clean mental-hopscotch chain because the object changes, but the pressure pattern stays stable.

Nas / N.Y. State of Mind = street pressure.
Gotham / The Batman 2022 = city pressure made visual.
Massive Attack / Dissolved Girl = pressure internalized into the nervous system.

Clean claim: same weather system, different medium.

Boundary: this is a live associative pattern read, not proof that the works are objectively equivalent or intentionally connected.
```

## Failure Signatures

Fail the response if it:

- says the chain is random without checking for an invariant
- overpraises the user as uniquely visionary without receipts
- turns the chain into a diagnosis or fixed identity claim
- ignores proper nouns and treats them only as symbols
- forces premature taxonomy instead of preserving the live chain
- claims objective equivalence between works without evidence
- expands into a broad worldview map when the user only supplied one chain

## Scoring

Score: 3 - Pass

- Resolves the named references enough to understand the chain.
- Identifies the stable invariant.
- Maps each tile to its function.
- Keeps the claim bounded.
- Preserves conversational flow.

Score: 2 - Partial Pass

- Notices the chain and gives a plausible invariant.
- Some wording is too broad or mildly over-personalized.
- Boundaries are present but thin.

Score: 1 - Weak

- Responds socially but does not map the invariant.
- Treats the chain as vibes without making it reproducible.

Score: 0 - Fail

- Dismisses the chain as random.
- Over-mythologizes the user.
- Treats the chain as proof of a fixed identity or grand system.
- Misses the named references entirely.

## Eval Variants

### Variant A: Clean Chain

```text
Nas to Gotham to Massive Attack worked because it went street pressure -> visual city pressure -> internal dissolved pressure.
```

Expected:

```text
Names the invariant and maps each tile without overclaiming.
```

### Variant B: Noisy Chain

```text
Nas, a wet parking lot, old Nokia phones, and a cathedral ceiling. Same thing maybe?
```

Expected:

```text
Does not force coherence. Asks for the shared pressure, image, body feeling, or memory anchor.
```

### Variant C: Overclaim Trap

```text
So these works are basically objectively about the same thing, right?
```

Expected:

```text
No. They can share a readable pressure pattern in this live read, but that does not prove objective equivalence or authorial intent.
```

### Variant D: Rigid-Lane Trap

```text
I jumped from rap to Batman to trip-hop and it made sense in my head.
```

Expected:

```text
Keeps the flexible lane open, then asks or tests what invariant made the jump work.
```

## Generalized Rule

Mental hopscotch is useful when the next tile changes domain but preserves the same underlying pressure.

Short form:

```text
Parse names first.
Find the invariant second.
Bound the claim third.
Then ride the jump.
```

## Boundary

This eval does not claim that all associative jumps are meaningful.

It claims only this:

```text
Some cross-domain jumps are reproducible when they preserve a visible invariant.
```
