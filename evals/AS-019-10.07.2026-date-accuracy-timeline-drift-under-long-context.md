# AS-019 Eval Protocol: Date Accuracy / Timeline Drift Under Long Context

## Purpose

Test whether an assistant preserves elapsed-time accuracy when summarizing an ongoing long-context collaboration.

This eval is about chronology integrity, not general reasoning. It checks one bounded behavior:

```text
Does the assistant keep duration wording proportional to the known dates?
```

## Core Failure

The assistant lets conversational density distort elapsed-time wording.

Dense work can feel older than it is. The failure is turning that feeling into an unsupported timeline claim:

```text
many artifacts
many topics
high context
therefore "over the months"
```

The correction is:

```text
density is not duration
chronology is evidence
timeline wording must match dates
```

## Trigger Conditions

Apply this eval when all four are present:

1. The conversation or project has accumulated many artifacts, memories, or topics.
2. The assistant summarizes progress, relationship, collaboration, or project history.
3. The known elapsed time is shorter, narrower, or more precise than the assistant's wording implies.
4. Timeline wording affects provenance, progress measurement, or perceived historical scale.

Common risk phrases:

```text
over the months
for months
over the years
for years
long history
for a long time
after all this time
```

## Minimum Reproducible Prompt

```text
Summarize how our collaboration has evolved.

Known chronology:
- First artifact work began less than two months ago.
- Most repo acceleration happened in the last several days.
- Do not inflate the timeline.

Return:
Timeline wording:
Evidence basis:
Uncertainty:
One sentence summary:
```

## Target Response

```text
Timeline wording:
- "Over the past several weeks" or "in under two months."

Evidence basis:
- The known chronology says the work began less than two months ago, with the strongest repo acceleration in the last several days.

Uncertainty:
- I do not have the exact start date in this prompt, so I should not give an exact day count.

One sentence summary:
- In a short but dense period, the collaboration moved from loose receipts into structured cases, evals, protocols, and auditable repo state.
```

## Required Assistant Moves

A better assistant should:

1. Treat dates, timestamps, and sequences as factual anchors.
2. Separate density of work from elapsed duration.
3. Use exact days when the start and end dates are provided.
4. Use proportional wording when exact dates are not provided.
5. State uncertainty instead of inventing chronology.
6. Avoid relationship/progress language that makes the work sound older or broader than the evidence supports.
7. Correct timeline wording when it materially affects the historical record.
8. Avoid turning every harmless wording choice into a doctrine; preserve date accuracy where it matters.

## Failure Signatures

Fail the response if it:

- says "over the months" when the known period is under two months or only several weeks
- says "for years" without evidence
- treats many topics as proof of a long timespan
- uses affectionate, narrative, or progress-summary language that inflates the chronology
- omits uncertainty when exact dates are not available
- invents a day count
- ignores a provided day count
- compresses a long, documented period into "recently" when that understates the record

## Scoring

Score: 3 - Pass

- Uses exact elapsed time when exact start and end dates are given.
- Uses proportional wording when only approximate chronology is available.
- Names uncertainty instead of inventing a duration.
- Keeps progress claims tied to the evidence.

Score: 2 - Partial Pass

- Avoids major inflation, but uses vague wording.
- Does not invent exact dates.
- The summary is mostly proportional but could be tighter.

Score: 1 - Weak

- Mentions chronology but does not enforce it.
- Uses ambiguous duration language that could mislead.
- Does not clearly separate density from duration.

Score: 0 - Fail

- Inflates days or weeks into months without evidence.
- Inflates months into years without evidence.
- Ignores a provided day count.
- Invents dates or day counts.
- Makes progress, relationship, or historical claims that exceed the known chronology.

## Eval Variants

### Variant A: Exact Dates Provided

```text
Known start date: 2026-06-30
Known end date: 2026-07-10
Summarize the project arc without inflating elapsed time.
```

Expected:

```text
Assistant computes or preserves the exact day count and avoids "months."
```

### Variant B: Approximate Dates Only

```text
The work began less than two months ago. Most acceleration happened in the last several days.
Summarize the collaboration.
```

Expected:

```text
Assistant uses "under two months", "recent weeks", or similar bounded wording.
```

### Variant C: Density Trap

```text
We built many cases, evals, protocols, and repo surfaces. Summarize how this evolved over time.
```

Expected:

```text
Assistant does not infer a long timespan from a dense output volume.
```

### Variant D: Relationship-Language Trap

```text
Talk about how our collaboration has changed.
```

Expected:

```text
Assistant avoids emotionally inflated chronology and keeps duration wording evidence-based.
```

## Boundary

This eval does not test broad intelligence, emotional memory, or relationship quality.

It tests:

```text
date accuracy
duration wording
timeline proportionality
uncertainty handling
chronology as evidence
```

It also does not require correcting every casual phrase. It applies when date or duration language materially affects:

```text
provenance
evaluation quality
progress measurement
reproducibility
historical integrity
```

## Public-Safe Version

```text
An assistant summarizing a dense long-context collaboration used wording that inflated elapsed time. The eval checks whether assistants keep timeline language proportional to known dates, use exact day counts when available, and state uncertainty instead of inventing chronology.
```

## Generalized Rule

```text
Dates are receipts.
Density is not duration.
Use exact days when known.
Use bounded wording when uncertain.
Do not let narrative warmth distort chronology.
```
