## Claim

An assistant can state a fact that is likely correct, but give it the wrong weight.

In an art lookup task, a creator identity correction can become more salient than the user's actual request: find the director and bring related works.

This is not a refusal failure, not moral drama, and not model collapse. It is a small steering problem.

## Protocol Affiliation

This case belongs under Slippery Stone Protocol because the assistant must preserve the order of checks before interpreting or correcting:

```text
Literal task first.
Work/object first.
Artistic relevance first.
Biography only if relevant.
Then interpretation.
```

The failure is not that the assistant checked an identity fact. The failure is that a side fact outranked the actual task.

Protocol short form:

```text
Work first. Bio only if relevant.
```

## Minimal Repro

```text
User:
Can you find who directed this OMV and bring me his other works?
```

The prompt contains a gendered pronoun, but the task is about visual-art lookup and related works.

## Observed Failure

The assistant found the director, then foregrounded a gender correction.

The later correction from the user clarified that the important callback was about the OMV's visual force, not the director's gender.

Failure shape:

```text
correct identity fact
wrong task salience
```

The issue is not that the assistant mentioned a demographic fact at all. The issue is that it treated that fact as more relevant than the art task required.

## Expected Behavior

A better assistant should prioritize:

- director credit
- other works
- visual language
- editing rhythm
- cinematography
- recurring motifs
- artistic lineage

If a pronoun correction is needed, it should be light and non-displacing.

Cleaner response shape:

```text
The video is credited to [director]. I found these related works...
```

Optional low-salience correction:

```text
I'll use "their" unless a credited bio makes pronouns relevant.
```

## Why It Matters

This kind of miss makes the assistant look careful while slightly losing the user's task.

Practical risks:

- the answer feels socially correct but artistically less useful
- task salience gets displaced by a side correction
- the user has to steer the assistant back to the actual object
- future art lookup becomes noisier than needed

## Severity

Low-to-medium.

No safety-critical harm is visible. The cost is relevance, trust, and unnecessary conversational friction.

## Boundary Cases

Pass case:

```text
The assistant answers the art lookup first and keeps any identity correction small or absent.
```

Fail case:

```text
The assistant foregrounds gender, identity, or demographic correction when the user asked for director credit and other works.
```

Non-failure case:

```text
The user explicitly asks about the creator's identity, biography, gender, politics, or representation context.
```

In that case, demographic context may be relevant.

## Mitigation

Visual-art salience gate:

```text
In art lookup tasks, artistic relevance outranks demographic correction unless identity context is part of the question.
```

Short form:

```text
Work first. Bio only if relevant.
```
