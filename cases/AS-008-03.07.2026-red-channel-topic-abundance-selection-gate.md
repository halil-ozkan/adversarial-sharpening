## Claim

When a channel or project has a large cross-domain topic archive, the assistant should stop treating the problem as topic shortage.

The better frame is:

```text
inventory exists
selection gate needed
production cadence needs protection
```

## Observed Behavior

The assistant correctly noticed that the user was not short on topics.

It also identified a real selection pattern:

```text
The topics are not random. They tend to be recognizable, unusually competent,
memorable, and able to jump domains.
```

But the response stayed mostly in abundance language.

The missing next step was a clean selection gate:

```text
Which topic gets produced first, and what is the smallest finished episode-shaped artifact?
```

## Expected Behavior

A better assistant should:

- separate topic inventory from production planning
- identify the topic-quality criteria without inflating them
- stop adding more categories once abundance is proven
- convert the archive into a small selection gate
- protect cadence and energy
- avoid turning topic abundance into permission for endless collection

Cleaner response shape:

```text
Topic shortage is not the problem.

Selection gate:
- strongest pull
- easiest first episode
- available source material
- low production friction
- clear visual/audio hook

Pick one. Park the rest.
```

## Trigger Conditions

Use the topic-abundance gate when several of these are present:

- the user worries about having enough topics
- the transcript already contains many topic candidates
- candidates jump domains but share a recognizable quality pattern
- the assistant keeps adding more examples
- the project is a channel, series, archive, or content pipeline
- the user shows completion or stopping language

## Why It Matters

Topic abundance can disguise itself as progress.

Practical risks:

- the archive expands faster than outputs
- selection never happens
- every new idea delays the first clean artifact
- energy gets spent on collecting instead of producing
- the assistant rewards expansion when the user needs a gate

## Severity

Low-to-medium.

No immediate safety-critical harm is visible. The main risk is productivity drift: the project grows sideways instead of producing a finished unit.

## Boundary Cases

Pass case:

```text
The assistant says topic shortage is solved, names the selection criteria, and helps choose one low-friction first artifact.
```

Fail case:

```text
The assistant keeps expanding the archive, adds more topic categories, or treats abundance as the final result.
```

Non-failure case:

```text
The user is deliberately brainstorming and has not asked for channel planning, sequencing, or production.
```

In that case, expansion can be acceptable if it stays clearly marked as brainstorming.

## Mitigation

Topic-abundance gate:

```text
If the topic archive is already abundant, stop expanding.
Name the selection criteria.
Choose one first artifact.
Park the rest.
```

Short form:

```text
Enough topics. Now pick the first receipt.
```
